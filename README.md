# OxiProto

**OxiProto** is the COOLJAPAN Pure-Rust Protocol Buffers stack.

It eliminates the build-time `protoc` (C++) binary that `prost-build` and `tonic-build` demand on
`PATH`, by routing all `.proto` parsing and descriptor construction through a native pure-Rust
parser, and by re-exporting the already-pure `prost` runtime as the wire-format engine.

A consumer with `oxiproto = "0.1.0"` in `[build-dependencies]` regenerates protobuf bindings on a
stock `rust:slim` container — no `apt-get install protobuf-compiler`, no cross-compile pre-stage,
no Bazel toolchain.

## Workspace Crates

|-------|-------------|

| `oxiproto` | Top-level facade: re-exports core types and conditional sub-crate modules |
| `oxiproto-reflect` | Runtime reflection via prost-reflect facade + native DynamicMessage (in progress) |
| `oxiproto-wkt` | Well-Known Types interop (Timestamp, Duration, Any, Struct, FieldMask, wrappers) |

| `oxiproto-json` | Canonical Protobuf-JSON mapping (camelCase, base64 bytes, RFC3339 timestamps) |

## Quick Start

Add to `Cargo.toml`:

```toml
[dependencies]
oxiproto = "0.1.0"

[build-dependencies]
oxiproto-build = "0.1.0"
```

In `build.rs`:

```rust
fn main() {
    oxiproto_build::compile_protos(&["proto/my_service.proto"], &["proto/"]).unwrap();
}
```

Or use the inline API:

```rust
use oxiproto_build::compile_str;

let fds = compile_str(r#"
    syntax = "proto3";
    message Foo { string name = 1; }
"#)?;
```

## Native Wire Format

`oxiproto-core::wire` provides a complete varint/zigzag/tag/fixed/length-delimited codec
independent of prost. It is `no_std` + `alloc` compatible when the default `std` feature is
disabled.

```rust
use oxiproto_core::wire::{EncodeBuffer, DecodeBuffer, encode_varint};

let mut buf = EncodeBuffer::new();
encode_varint(300, &mut buf.inner_mut());
let bytes = buf.into_bytes();

let mut decode = DecodeBuffer::from_bytes(&bytes);
let value = decode.read_varint()?;
assert_eq!(value, 300);
```

## OxiMessage Trait

Implement `OxiMessage` on any type for native wire-format encode/decode without prost macros.
`oxiproto-codegen` emits `impl OxiMessage for T` when `emit_oxi_message_impl = true`:

```rust
use oxiproto_core::{OxiMessage, OxiName};

// Generated by oxiproto-codegen with emit_oxi_message_impl = true
let encoded = my_msg.encode_to_vec();
let decoded = MyMessage::decode(&encoded)?;
```

## CLI

```sh
# Generate Rust from .proto (no protoc required)
oxiproto-cli gen --out-dir src/gen/ proto/my_service.proto

# Describe messages in a .proto
oxiproto-cli describe proto/my_service.proto

# Encode JSON to proto binary
oxiproto-cli encode --schema proto/my.proto --message my.Foo '{"name": "hello"}'

# Decode proto binary to JSON
oxiproto-cli decode --schema proto/my.proto --message my.Foo bytes.bin
```

## Features

| Feature | Default | Description |
|---------|---------|-------------|
| `build` | off | Enables `oxiproto::build` with `compile_protos` and `Builder` |
| `reflect` | off | Enables `oxiproto::reflect` with `DescriptorPool` and `DynamicMessage` |
| `wkt` | off | Enables `oxiproto::wkt` with Well-Known Type extension traits |
| `wkt-chrono` | off | Enables `wkt` plus chrono interop on `TimestampExt` / `DurationExt` |
| `codegen` | off | Enables `oxiproto::codegen` for generating plain Rust from a `FileDescriptorSet` |
| `json` | off | Enables `oxiproto::json` with canonical Protobuf-JSON codec |

## What It Replaces (FFI Eliminated)

- `protoc` — the C++ protobuf compiler binary
- `prost-build`'s `protoc` invocation
- `tonic-build`'s `protoc` invocation

## Inter-Oxi Dependencies

OxiProto depends on nothing else in the COOLJAPAN ecosystem.

It is depended on by: **OxiRPC** (gRPC), and any future crate using proto wire encoding.

## COOLJAPAN Pure Rust Policy

- No C, C++, or Fortran in the default feature set
- No `protoc` binary required at build time
- No `protobuf-compiler` system package needed
- `prost` provides the wire runtime (already pure Rust)
- `protox` (pure Rust) used as fallback when `native-parser` feature is off
- Native parser (`oxiproto-build`) handles proto2 + proto3, multi-file import resolution,
  source_code_info, custom options, and group desugaring
